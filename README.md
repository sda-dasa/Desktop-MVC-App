Чтобы использовать шаблон, необходимо форкнуть его себе, нажав на кнопку Fork в правом верхнем углу.

ДЛЯ ТОГО, ЧТОБЫ СИСТЕМА РАБОТАЛА, ВЫПОЛНИТЬ СЛЕДУЮЩИЕ ШАГИ ДО ПЕРВОГО КОММИТА: 
1) Создать персональный токен в Edit profile -> Access Tokens, выдав ему только права на read_repository и на write_repository и установив время его жизни до конца семестра.  
2) Далее в проекте, который вы получили после форка шаблонного репозитория, надо перейти в Settings -> CI/CD -> Variables.  
3) Полученный токен добавить в переменную PRIVATE_TOKEN (с флагами Masked и Expanded, без Protected), а в GITLAB_USERNAME (выключить все флаги) добавить свой... юзернейм!

Для того, чтобы запустить тесты, изменения необходимо закоммитить в ветку test, используя правильные названия коммитов.  
Папка для кода - /src, ее надо самостоятельно создать.
 
Формат названий коммитов:  
L<номер лабы> T<номер задания> [V<номер варианта>] [комментарий (по желанию)]  
Примеры:  
L1 T1 hello world  
L1 T2.2 V5 some bugs fixed  
L2 T2 V12  

Все изменения происходят в ветке test - код, пайплайны с тестами, отчеты по тестам. В ветку main можно замерджить изменения после выполнения лабораторной и прохождения тестов. 

Если что-то не работает, проверьте:
1) Правильно ли назван коммит;
2) В правильную ли ветку вы пушите;
3) Есть ли обновления в оригинальном репозитории. Для этого проверьте, есть ли кнопка Update fork на странице репозитория в GitLab.
